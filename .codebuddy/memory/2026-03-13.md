# 2026-03-13 工作日志

## 搭建后端 Express + SQLite 服务

为玺华铝业业务管理系统新增后端服务，实现订单数据的持久化存储。

### 后端结构（`server/` 目录）
- **技术栈**：Express + better-sqlite3 + tsx
- **端口**：3456
- **数据库**：SQLite，文件存放在 `server/data/xihualvye.db`
- **API 接口**：
  - `POST /api/orders` — 创建订单
  - `GET /api/orders` — 查询订单列表（支持分页、按客户/单号/款式筛选）
  - `GET /api/orders/:id` — 获取订单详情
  - `PUT /api/orders/:id` — 更新订单
  - `DELETE /api/orders/:id` — 删除订单
  - `POST /api/orders/batch-delete` — 批量删除

### 前端改造
1. 新增 `src/ts/api.ts` — 前端 API 请求封装
2. 修改 `button2.ts` — 保存时同时保存到后端和 localStorage（localStorage 作为备份）
3. 新增 `loadOrderFromServer()` 和 `setCurrentOrderId()` 函数
4. 所有 7 个 vue 页面的 `onMounted` 修改为支持从后端加载指定订单（通过 URL query `?orderId=xxx`）
5. 新增 `src/views/OrderManage.vue` — 订单管理页面（列表查询、详情查看、加载到编辑页、删除、批量删除）
6. 路由菜单中新增"订单管理"入口
7. `vite.config.ts` 添加 `/api` 代理到 `localhost:3456`

### 启动方式（开发环境）
- 后端：`cd server && npm run dev`
- 前端：`cd xihualvye && npm run dev`
- 先启动后端，再启动前端

## 生产环境部署改造

### 改造内容
1. **`server/src/index.ts`** — 生产模式下自动托管 `server/public/` 目录的前端静态文件，支持 Vue Router history 模式，监听 `0.0.0.0` 允许外网访问，端口通过 `PORT` 环境变量可配
2. **`server/src/db.ts`** — 数据库路径支持 `DB_DIR` 环境变量自定义
3. **`build.bat`** / **`build.sh`** — 一键打包脚本：打包前端 → 复制到 server/public → 安装依赖

### 部署方式
- 前端打包后放入 `server/public/`
- 只部署 `server/` 文件夹到服务器
- 运行 `npm start`（即 `node dist/index.js`）或用 PM2：`pm2 start "node dist/index.js" --name xihualvye`
- 前后端由同一个 Express 服务托管，只占一个端口

## 兼容 Node.js v16 改造

### 原因
服务器 CentOS 7.6 上 Node.js 版本为 v16.20.2 且不能升级。原项目使用 ESM + tsx 运行 TypeScript，不兼容 Node 16。

### 改造内容
1. **`server/tsconfig.json`** — 改为 `module: CommonJS`, `target: ES2021`, `moduleResolution: node`
2. **`server/package.json`** — 去掉 `"type": "module"`；`start` 脚本改为 `node dist/index.js`（运行编译后的 JS）；数据库从 `better-sqlite3` 换成纯 JS 的 `sql.js`（不需要编译原生模块）
3. **`server/src/db.ts`** — 用 `sql.js` 重写，提供与 better-sqlite3 兼容的 prepare/run/get/all API；异步初始化，提供 `waitForReady()` 方法
4. **`server/src/index.ts`** — 去掉 ESM 的 `import.meta.url`（CommonJS 下 `__dirname` 可用）；启动前 `await db.waitForReady()`
5. **`server/src/routes/orders.ts`** — import 路径去掉 `.js` 后缀
6. **`build.bat` / `build.sh`** — 新增第 3 步：编译后端 TypeScript（`npx tsc`）

### 新的部署流程
1. 本地：双击 `build.bat`（打包前端 + 编译后端）
2. XFTP 上传 `server/` 下的 `dist/`、`public/`、`package.json`、`package-lock.json`（跳过 `data/` 和 `node_modules/`）
3. 服务器：`cd /xmc/xihualvye && npm install --production && pm2 restart xihualvye`
