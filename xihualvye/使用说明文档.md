# 玺华铝业 - 业务(表格)管理系统 使用说明文档

---

## 一、环境要求

在运行本项目之前，请确保已安装以下环境：

| 环境 | 最低版本要求 | 推荐版本 |
|------|------------|---------|
| **Node.js** | `>=20.19.0` 或 `>=22.12.0` | v22.20.0 |
| **pnpm**（包管理器） | `>=8.0.0` | v9.x |
| **npm**（备选） | `>=8.0.0` | v10.x |

### 1.1 安装 Node.js

前往 [Node.js 官网](https://nodejs.org/) 下载并安装 LTS 版本（推荐 v22.x）。

安装完成后，打开终端验证：

```bash
node --version
# 输出示例：v22.20.0
```

### 1.2 安装 pnpm（推荐的包管理器）

本项目使用 **pnpm** 管理依赖（项目中存在 `pnpm-lock.yaml` 文件）。

```bash
npm install -g pnpm
```

安装完成后验证：

```bash
pnpm --version
# 输出示例：9.15.9
```

> **备注**：如果不想安装 pnpm，也可以使用 npm，但推荐使用 pnpm 以保证依赖版本一致性。

## 三、运行项目

### 3.1 开发模式运行（本地开发调试）

```bash
pnpm dev
```

或使用 npm：

```bash
npm run dev
```

运行成功后，终端会显示本地访问地址，例如：

```
  VITE v7.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: http://192.168.x.x:5173/
```

在浏览器中打开 `http://localhost:5173/` 即可访问系统。

### 3.2 构建生产环境版本

```bash
pnpm build
```

或使用 npm：

```bash
npm run build
```

构建完成后，产物会输出到 `dist/` 目录，可直接部署到静态服务器。

### 3.3 预览生产构建

```bash
pnpm preview
```

或使用 npm：

```bash
npm run preview
```

此命令会启动一个本地服务器来预览 `dist/` 目录中的构建产物。

### 3.4 代码格式化

```bash
pnpm format
```

或使用 npm：

```bash
npm run format
```

## 六、系统功能说明

### 6.1 整体架构

本系统是一个基于 **Vue 3 + TypeScript + Vite + Element Plus** 的单页面应用（SPA），用于玺华铝业的业务表格管理。系统采用左侧菜单导航 + 右侧内容区的经典后台布局。

### 6.2 页面功能

系统目前包含 **天枢款** 系列共 6 个子页面：

| 菜单项 | 路由路径 | 说明 |
|--------|---------|------|
| 天枢款-单门-加背板 | `/tianshu/a` | 单门加背板款式的表格管理 |
| 天枢款-单门-加背板-加固 | `/tianshu/b` | 在单门加背板基础上增加加固项 |
| 天枢款-常用款 | `/tianshu/c` | 常用款式的表格管理 |
| 天枢款-无上包边款 | `/tianshu/d` | 无上包边款式的表格管理 |
| 天枢款-双面门款 | `/tianshu/e` | 双面门款式的表格管理 |
| 天枢款-双面门款-背面假门 | `/tianshu/f` | 双面门背面假门款式的表格管理 |

### 6.3 每个页面的核心功能

每个页面都包含以下功能模块：

1. **基本信息区**：输入客户名称、日期、表面处理（下拉选择：冰川白/米兰灰/幻夜黑）、数量、客户单号、长宽高尺寸
2. **主数据表格**：展示型号、图片、名称、规格、数量、备注，支持行合并显示，图片来源于 `JPG/` 目录
3. **底部门板数据**：门板、侧门板、背板的相关数据
4. **配件表**：上包转角、下包转角、三卡锁、堵头、角码、铰链等配件数据
5. **工艺备注**：自由文本备注区域
6. **门数量/中柱数量**：在表格右侧浮动显示的快捷输入

### 6.4 操作按钮功能

每个页面顶部有三个操作按钮：

| 按钮 | 功能说明 |
|------|---------|
| **下载表格** | 将当前页面所有表格数据导出为 `.xlsx` 文件，包含产品图片嵌入 |
| **保存表格数据** | 将当前页面所有数据保存到浏览器本地存储（localStorage），防止切换页面数据丢失 |
| **清空表格** | 清空当前页面所有已填写的数据，恢复默认值（需二次确认） |

### 6.5 数据自动保存

- 页面打开时会自动从 localStorage 加载之前保存的数据
- 门板数据和工艺备注修改时会自动触发保存

## 七、常见问题

### Q1: 安装依赖报错怎么办？

检查 Node.js 版本是否满足要求（`>=20.19.0` 或 `>=22.12.0`）：

```bash
node --version
```

如果版本不符，请升级 Node.js。

### Q2: 运行 `pnpm dev` 后浏览器无法访问？

1. 确认终端输出的端口号（默认 5173）
2. 检查是否有防火墙或杀毒软件拦截
3. 尝试直接访问 `http://localhost:5173/`

### Q3: 下载的 xlsx 文件中没有图片？

确保 `JPG/` 目录下存在对应型号的图片文件（如 `H-1001.jpg`），图片文件名需与表格中的型号字段匹配。

### Q4: 切换页面后数据丢失了？

请在切换页面前点击 **"保存表格数据"** 按钮，将数据保存到本地存储。下次进入该页面时会自动恢复。
