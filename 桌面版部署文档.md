# 玺华铝业桌面版 - 完整部署文档

## 一、整体架构

```
┌──────────────┐         ┌──────────────┐
│  客户1 exe    │──┐     │              │
├──────────────┤  │      │  中央服务器    │
│  客户2 exe    │──┼─────►│  Node.js后端  │
├──────────────┤  │      │  SQLite 数据库│
│  客户3 exe    │──┘     │              │
└──────────────┘         └──────────────┘
     用户电脑              (局域网/云服务器)
```

**数据存储**：所有业务数据（订单、客户、状态等）都存在**中央服务器**的 SQLite 数据库，多个客户端共享，实时同步。

## 二、打包流程

### 前置条件
- Windows 10/11
- 已安装 Node.js **20+**
- 本地代码是最新的

### 一键打包（三步）

#### 步骤 1：打包前端 + 后端
```bat
build.bat
```
产物：`server/dist/` 和 `server/public/`

#### 步骤 2：部署服务器到中央机器
参考 `部署上线文档.md`，把 `server/dist/`、`server/public/`、`package.json`、`package-lock.json` 上传到服务器，然后 `npm install --production && pm2 start dist/index.js --name xihualvye`。

服务器地址是 `http://42.194.214.100:3456`。

#### 步骤 3：打包桌面客户端
```bat
build-electron.bat
```

首次运行会安装 Electron 依赖（~150MB，需要几分钟）。后续打包秒级完成。

产物：`electron/release/玺华铝业-Setup-1.0.0.exe`（约 80-120 MB）

## 三、分发给用户

直接把 `玺华铝业-Setup-1.0.0.exe` 发给用户（微信/U盘/OA），用户操作：

1. 双击安装程序
2. 选择安装目录（默认用户目录，无需管理员权限）
3. 安装完成 → 桌面出现"玺华铝业"图标
4. 首次启动：弹出配置页，输入 `http://192.168.1.100:3456` → 点击"测试连接" → 连接成功 → 点击"保存并进入系统"
5. 进入登录页，输入账号密码（`xhly / 123123` 或管理员配置的账号）
6. 之后每次双击图标直接进入系统

## 四、自动更新机制

### 搭建更新服务器（选 nginx 或云存储）

#### 方案 A：用现有 nginx 服务器
```nginx
server {
    listen 80;
    server_name updates.your-domain.com;

    location /xihualvye-updates/ {
        alias /var/www/xihualvye-updates/;
        autoindex off;
    }
}
```

上传目录结构：
```
/var/www/xihualvye-updates/
├── latest.yml                           # electron-builder 自动生成
├── 玺华铝业-Setup-1.0.1.exe
└── 玺华铝业-Setup-1.0.1.exe.blockmap
```

#### 方案 B：用阿里云 OSS / 腾讯云 COS
创建公开读的 Bucket，把以上三个文件上传进去。

### 修改 electron 配置指向你的更新服务器
编辑 `electron/electron-builder.yml`：
```yaml
publish:
  provider: generic
  url: https://updates.your-domain.com/xihualvye-updates/
  channel: latest
```

### 发布新版本流程

1. 修改 `electron/package.json` 的 `version`（如从 `1.0.0` → `1.0.1`）
2. 执行 `build-electron.bat`
3. 把 `electron/release/` 下的这三个文件上传到更新服务器：
   - `latest.yml`
   - `玺华铝业-Setup-1.0.1.exe`
   - `玺华铝业-Setup-1.0.1.exe.blockmap`
4. 用户客户端 30 秒内自动检测到新版，后台下载，下载完成弹窗提示重启

## 五、服务器端升级（不需要用户配合）

如果只是业务逻辑改动（不涉及桌面客户端代码），只需要更新服务器：

```bash
build.bat                                 # 本地打包
# 上传 server/dist + server/public 到服务器
pm2 restart xihualvye                     # 服务器重启
```

用户下次打开客户端会自动拿到新界面，**无需更新 exe**。

## 六、用户数据与密码

### 登录系统
- 保留了原有登录机制
- 默认账号：`xhly / 123123`
- 建议部署后通过服务器环境变量修改：
  ```bash
  ADMIN_USERNAME=myadmin ADMIN_PASSWORD=StrongPwd123 pm2 restart xihualvye --update-env
  ```

### 用户凭据储存位置（每台客户端）
- 服务器地址：`%APPDATA%\玺华铝业业务管理系统\config.json`
- 登录 token：浏览器 localStorage（由 Electron 管理）

### 业务数据
**全部在中央服务器的 SQLite 数据库中**，任何客户端的订单保存都会实时写入中央库。其他客户端刷新即可看到最新数据。

## 七、常见问题

### Q1. 用户客户端连不上服务器？
1. 确认中央服务器运行正常：浏览器访问 `http://42.194.214.100:3456` 能不能打开
2. 确认防火墙放行 3456 端口
3. 让用户右键托盘 → 配置服务器地址 → 测试连接
4. 若是云服务器，确认安全组已开 3456

### Q2. 内网穿透 / 异地办公怎么办？
- 方案 A：公司拉专线 / VPN
- 方案 B：中央服务器放在云端（公网 IP 或域名）
- 方案 C：用 [frp](https://github.com/fatedier/frp) 做内网穿透

### Q3. 自动更新失败？
- 检查客户端日志：`%APPDATA%\玺华铝业业务管理系统\logs\`
- 确认更新服务器的 `latest.yml` 可通过浏览器访问
- 确认 `latest.yml` 中的版本号与实际 exe 文件名一致

### Q4. 数据库备份？
参考 `部署上线文档.md` 的"数据库备份"章节，crontab 自动备份 `server/data/xihualvye.db`。

### Q5. Windows Defender 提示"未知发布者"？
未做代码签名的 exe 会被警告。解决：
- 申请代码签名证书（商业证书 ~¥500-2000/年）
- 或让用户点击"更多信息" → "仍要运行"（一次性）

### Q6. 想让某些电脑只能查看，不能修改？
当前只有一个账号，需要扩展权限系统。如果需要，告诉我可以加上"只读/管理员"两种角色。

- 卸载：控制面板 → 卸载玺华铝业业务管理系统
- 默认会**保留**用户配置（服务器地址等），重装后直接能用
