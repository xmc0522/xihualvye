| 工具 | 现在版本 | 用途 |
|------|---------|------|
| Node.js | **20.18** | 运行 npm / vite / tsc |
| npm | 10.8.2 | 包管理 |

### 服务器（部署目标）
| 工具 | 现在版本 | 用途 |
|------|---------|------|
| Linux/CentOS/Ubuntu | 任意主流版本 | 操作系统 |
| Node.js | **20.18** | 运行后端服务 |
| PM2 | 7.0.1 | Node 进程守护 |
| nginx（可选） | 1.26.3 | 反向代理 / HTTPS |

> 后端使用了纯 JS 的 `sql.js`，**不依赖系统 SQLite 库**，对服务器零依赖，开箱即可运行。

### 服务器初始化（首次部署时执行一次）

```bash
# 刷新系统缓存
dnf makecache

# 安装基础依赖
dnf install -y wget curl

# 下载 Node.js 20
wget https://npmmirror.com/mirrors/node/v20.18.0/node-v20.18.0-linux-x64.tar.xz

# 解压
tar -xf node-v20.18.0-linux-x64.tar.xz -C /usr/local/

# 重命名方便管理
mv /usr/local/node-v20.18.0-linux-x64 /usr/local/node

# 配置环境变量（永久生效）
echo "export PATH=/usr/local/node/bin:\$PATH" >> /etc/profile

# 立刻生效
source /etc/profile

node -v
npm -v

# 安装 PM2
npm install -g pm2 --registry=https://registry.npmmirror.com

# 验证 PM2
pm2 -v

# 配置 PM2 开机自启（服务器重启不掉线）
pm2 startup
pm2 save
```

## 三、首次部署完整流程

### Step 1：本地打包

#### Windows
```bat
:: 进入项目根目录
cd C:\path\to\xihualvye

:: 双击或命令行执行
build.bat
```

#### Linux / Mac
```bash
cd /path/to/xihualvye
chmod +x build.sh
./build.sh
```

执行完成后，会生成两个关键产物：
- `server/public/`     — 前端静态文件
- `server/dist/`       — 后端编译产物

### Step 2：上传文件到服务器

使用 **XFTP / WinSCP / FileZilla** 上传到服务器目录（以 `/xmc/xihualvye/` 为例）：

| 必传 | 路径 | 说明 |
|:---:|------|------|
| ✅ | `server/dist/` | 后端 JS |
| ✅ | `server/public/` | 前端静态文件 |
| ✅ | `server/package.json` | 依赖清单 |
| ✅ | `server/package-lock.json` | 依赖锁定文件（保证版本一致） |

| 不传 | 路径 | 说明 |
|:---:|------|------|
| ❌ | `server/data/` | 服务器上的生产数据库，**覆盖会丢全部订单数据** |
| ❌ | `server/node_modules/` | 服务器上要重新安装（环境差异） |
| ❌ | `server/src/` | 源代码服务器不需要 |

> **首次部署时**：服务器上还没有 `data/` 目录，无需担心覆盖。后续更新时严禁上传本地的 `data/`。

### Step 3：服务器安装依赖 & 启动

```bash
ssh user@your-server-ip
cd /xmc/xihualvye

# 安装生产依赖（首次或 package.json 有变化时）
npm install --production

# 首次启动用 pm2
pm2 start dist/index.js --name xihualvye

# 保存当前进程列表（重启服务器后自动恢复）
pm2 save

# 查看日志
pm2 logs xihualvye --lines 50
```

预期看到日志：
```
📁 数据库路径: /xmc/xihualvye/data/xihualvye.db
✅ 数据库初始化完成
📦 生产模式：已加载前端静态文件
✅ 玺华铝业后端服务已启动: http://localhost:3456
```

### Step 4：访问验证

浏览器打开 `http://your-server-ip:3456` → 看到登录页 → 输入账号 `xhly / 123123` → 进入系统首页 → 见到统计卡片说明部署成功。

## 四、日常更新发布流程（已部署后）

### 每次发版的标准动作

#### 1. 本地打包
```bat
build.bat        :: Windows
./build.sh       # Linux/Mac
```

#### 2. 上传更新文件
仅上传变化的文件夹即可（**绝对不要上传 `data/`**）：
- `server/dist/`     （后端代码改动时）
- `server/public/`   （前端代码改动时）
- `server/package.json` + `server/package-lock.json`（添加/更新依赖时）

#### 3. 服务器重启
```bash
cd /xmc/xihualvye

# 仅当 package.json 有变化时执行
npm install --production

# 重启 PM2 服务（无缝热重启）
pm2 restart xihualvye

# 查看是否启动正常
pm2 logs xihualvye --lines 20
```

#### 4. 浏览器强制刷新
按 **Ctrl + F5**（或 Cmd+Shift+R）强制清缓存，确保加载到新前端代码。

## 五、数据库说明

### 数据存储位置
- 路径：`server/data/xihualvye.db`
- 格式：SQLite 单文件（由 `sql.js` 管理，无需服务器安装 SQLite）
- 大小：通常几 MB ~ 几十 MB，订单数 1000 条以内可忽略

### 自动迁移机制（重要）
本系统**已内置自动迁移**：每次启动时检查表结构，若发现旧数据库缺少新字段（如 `status`），会自动 `ALTER TABLE` 添加，**无需人工干预**。

> 升级到 v1.1 后首次启动，控制台会打印：
> `🔧 已为现有数据库自动添加 status 列`
> 之后所有历史订单的状态默认为"待生产"。

### 备份建议（极重要）

**强烈建议设置定时备份**，否则磁盘损坏 = 数据全丢。

#### 方案 A：crontab 自动备份（推荐）
```bash
# 编辑定时任务
crontab -e

# 添加：每天凌晨 2 点备份一次，保留近 30 天
0 2 * * * cp /xmc/xihualvye/data/xihualvye.db /xmc/backup/xihualvye_$(date +\%Y\%m\%d).db && find /xmc/backup -name "xihualvye_*.db" -mtime +30 -delete
```

#### 方案 B：手动备份
```bash
cp /xmc/xihualvye/data/xihualvye.db /xmc/backup/xihualvye_$(date +%Y%m%d).db
```

### 数据恢复
若需恢复某天的备份：
```bash
pm2 stop xihualvye
cp /xmc/backup/xihualvye_20260401.db /xmc/xihualvye/data/xihualvye.db
pm2 start xihualvye
```

## 八、可选：用 nginx 反向代理 + HTTPS

若需要域名访问 + HTTPS（强烈推荐生产环境使用）：

### 1. 安装 nginx
```bash
dnf install -y nginx
```

### 2. 配置反向代理
执行命令创建文件：

```nginx
cat > /etc/nginx/conf.d/xihualvye.conf << EOF
server {
    listen 80;
    server_name 42.194.214.100;

    # 第二个项目
    location /wmj/ {
        alias /xmc/wmj/dist/;
        index index.html;
        try_files \$uri \$uri/ /wmj/index.html;
    }

    # 主项目
    location / {
        root /xmc/xihualvye/frontend;
        index index.html;
        try_files \$uri \$uri/ /index.html;
    }

    # 后端API
    location /api/ {
        proxy_pass http://127.0.0.1:3456/api/;
        proxy_set_header Host \$host;
        proxy_set_header X-Real-IP \$remote_addr;
        proxy_set_header X-Forwarded-For \$proxy_add_x_forwarded_for;
    }

    # 静态资源
    location /public/ {
        proxy_pass http://127.0.0.1:3456/public/;
    }
}
EOF
```

### 3. 测试并重启
```bash
nginx -t       # 测试配置
systemctl restart nginx        # 重启 Nginx 生效
systemctl enable nginx         # 设置开机自启（防止重启服务器挂掉）
```

### 4. 申请免费 HTTPS 证书（强烈推荐）
```bash
# 安装 certbot
sudo yum install -y certbot python3-certbot-nginx

# 一键申请并自动配置 nginx HTTPS
sudo certbot --nginx -d xhly.example.com
```
之后访问 `https://xhly.example.com` 即可，证书 90 天后自动续期（certbot 自带定时任务）。

## 十、常见问题排查

### Q1. 启动报错 "Cannot find module 'xxx'"
原因：依赖未安装或不全。
```bash
cd /xmc/xihualvye
rm -rf node_modules
npm install --production
pm2 restart xihualvye
```

### Q2. 浏览器访问空白
1. 检查浏览器控制台报错 F12
2. 确认上传了 `server/public/` 目录
3. 检查 nginx 配置是否正确（如使用代理）
4. 强制刷新 Ctrl+F5

### Q3. 数据库文件锁住 / "database is locked"
极少见。如果出现：
```bash
pm2 stop xihualvye
# 等 5 秒确保完全停止
pm2 start xihualvye
```

### Q4. 升级后页面打不开
1. 确认强制刷新清浏览器缓存
2. `pm2 logs xihualvye` 看后端是否启动成功
3. 检查上传是否完整（`server/public/index.html` 必须存在）

### Q5. 想看实时性能
```bash
pm2 monit
```

### Q6. 服务器重启后服务没自动起来
```bash
pm2 startup    # 查看提示并执行其输出的命令
pm2 save       # 保存当前服务列表
```

## 十一、回滚方案（出问题时）

### 紧急回滚到上一版本

#### 前提：发版前先备份
```bash
# 发版前在服务器执行（强烈建议加入发版 SOP）
cp -r /xmc/xihualvye/dist /xmc/backup/dist_$(date +%Y%m%d_%H%M%S)
cp -r /xmc/xihualvye/public /xmc/backup/public_$(date +%Y%m%d_%H%M%S)
```

#### 回滚操作
```bash
cd /xmc/xihualvye
rm -rf dist public
cp -r /xmc/backup/dist_20260430_120000 dist
cp -r /xmc/backup/public_20260430_120000 public
pm2 restart xihualvye
```
