## 📖 阅读须知（先看这个，再开始）

### 1. 涉及哪两台电脑？

部署过程中你会用到 **两台电脑**，请先在脑子里把它们分清楚：

| 简称 | 是什么 | 干什么 |
|---|---|---|
| **本地电脑** | 你正在用的这台 Windows 电脑 | 写代码 + 打包 |
| **服务器** | 远程的 Linux 主机（云服务器） | 给用户访问的"网站"实际跑在这里 |

> 🟦 后续每段命令我都会标记 **【本地电脑】** 还是 **【服务器】**，请认准再敲。

### 2. 需要哪些工具？

| 软件 | 装在哪 | 用途 | 没有怎么办 |
|---|---|---|---|
| Node.js 20.18+ | 本地电脑 + 服务器 | 跑 npm 和后端 | 去 https://nodejs.org 下载 |
| FinalShell（或 Xshell） | 本地电脑 | 远程登录服务器、敲命令 | 推荐 FinalShell（免费、自带文件传输） |
| WinSCP（或 FileZilla） | 本地电脑 | 把打包好的文件上传到服务器 | FinalShell 也自带传输，可以不装 |

### 3. 需要哪些信息？

部署前请向运维或老板要到下面 4 个信息（缺一不可）：

```
服务器 IP：       例如 39.108.123.45
SSH 端口：        通常是 22
登录用户名：       通常是 root
登录密码：         xxxxxx
```

## 📁 一、项目结构速查

```
xihualvye/                       ← 项目根目录（在你本地电脑上）
├── xihualvye/                   ← 前端代码（Vue）
├── server/                      ← 后端代码（Node）
│   ├── src/                     ← 源码（不上传服务器）
│   ├── data/                    ← 数据库文件（不上传服务器！会覆盖线上数据！）
│   ├── public/                  ← 打包后的前端文件（要上传）
│   ├── dist/                    ← 打包后的后端文件（要上传）
│   ├── package.json             ← 依赖清单（要上传）
│   ├── package-lock.json        ← 依赖锁定（要上传）
│   └── ecosystem.config.js      ← PM2 启动配置（要上传）
├── electron/                    ← 桌面端（不影响网页部署，可忽略）
├── build.bat                    ← Windows 一键打包脚本
└── 部署上线文档.md              ← 本文档
```

🚨 **三条红线（必须背下来）**：
1. ❌ 不要把本地的 `server/data/` 上传服务器（会覆盖订单数据）
2. ❌ 不要把本地的 `server/node_modules/` 上传服务器（Windows 的二进制 Linux 跑不了）
3. ❌ 不要把本地的 `server/src/` 上传服务器（源码不需要）

> 如果你的服务器之前已经跑过这个系统，**直接跳到第二部分**。

## 二、登录服务器

### Step 1：打开 FinalShell，新建连接

1. 启动 FinalShell
2. 左上角点 "📁" → "SSH 连接（Linux）"
3. 填写信息：
   - 名称：随便填，比如 `xihualvye-prod`
   - 主机：你的服务器 IP
   - 端口：22
   - 用户名：root
   - 密码：xxxxxx
4. 点"确定"，然后双击连接

### Step 2：连接成功后看到类似下面的画面

```
[root@iZbp1abc ~]#
```

光标在这里闪烁，就说明你已经"进入服务器"了。**接下来的命令都在这个黑窗口里执行**。

## 三、安装基础环境（在服务器上执行）

> 🟧 **【服务器】** 把下面的命令一段一段复制到 FinalShell 里，按回车执行。

### Step 1：安装编译工具（防止后面装数据库失败）

```bash
# 如果是 CentOS / Rocky / 阿里云 Linux：
sudo dnf install -y gcc gcc-c++ make wget curl

# 如果是 Ubuntu / Debian：
# sudo apt-get update && sudo apt-get install -y build-essential wget curl
```

> ❓ 怎么知道我的服务器是 CentOS 还是 Ubuntu？敲一行 `cat /etc/os-release` 看输出里的 `NAME` 字段。

### Step 2：安装 Node.js 20

```bash
cd /tmp
wget https://npmmirror.com/mirrors/node/v20.18.0/node-v20.18.0-linux-x64.tar.xz
tar -xf node-v20.18.0-linux-x64.tar.xz -C /usr/local/
mv /usr/local/node-v20.18.0-linux-x64 /usr/local/node
echo "export PATH=/usr/local/node/bin:\$PATH" >> /etc/profile
source /etc/profile
```

**验证安装成功**：
```bash
node -v     # 应输出 v20.18.0
npm -v      # 应输出 10.x.x
```

如果输出"command not found"，重新执行一次 `source /etc/profile` 或重新登录服务器。

### Step 3：配置 npm 国内镜像（不配会很慢）

```bash
npm config set registry https://registry.npmmirror.com
```

### Step 4：安装 PM2（让 Node 服务在后台常驻）

```bash
npm install -g pm2
pm2 -v      # 应输出版本号，如 5.4.x
```

### Step 5：配置 PM2 开机自启

```bash
pm2 startup
```

执行后它会输出**一条很长的命令**，把它整条复制下来，粘贴回去再执行一次。比如它输出：
```
sudo env PATH=$PATH:/usr/local/node/bin /usr/local/node/lib/node_modules/pm2/bin/pm2 startup systemd -u root --hp /root
```
你就把上面这一整行再粘贴回 FinalShell 执行一次。

### Step 6：开放 3456 端口

#### 服务器自身防火墙（CentOS/Rocky）
```bash
sudo firewall-cmd --permanent --add-port=3456/tcp
sudo firewall-cmd --reload
```

#### 云服务器安全组（重要！别忘了）
登录你的云控制台（阿里云/腾讯云/华为云）：
1. 进入"云服务器 ECS" → 找到你这台机器 → "安全组"
2. 添加一条入站规则：协议 `TCP`，端口 `3456`，源 `0.0.0.0/0`
3. 保存

> ⚠️ 这一步**经常被忘**，导致部署完了浏览器访问不通。如果你后面访问页面打不开，先回来检查这一步。

### Step 7：创建项目目录

```bash
mkdir -p /xmc/xihualvye
mkdir -p /xmc/xihualvye/logs
mkdir -p /xmc/backup
```

✅ 至此服务器初始化完成。**这一部分一辈子只要做一次**。

# 第二部分：每次部署都要做的事

## 四、本地打包（在你的 Windows 电脑上执行）

> 🟦 **【本地电脑】**

### Step 1：双击 `build.bat`

在文件资源管理器里进入 `C:\Users\v_xmcoxu\Desktop\xihualvye\` 目录，**双击 `build.bat`** 文件。

会弹出一个黑色窗口，自动跑前端和后端的打包，**全程别关，等它跑完**（第一次约 3-5 分钟）。

### Step 2：检查产物

打包完成后，确认下面两个文件夹都有东西：

```
C:\Users\v_xmcoxu\Desktop\xihualvye\server\dist\        ← 后端编译产物（应该有 index.js 等）
C:\Users\v_xmcoxu\Desktop\xihualvye\server\public\      ← 前端静态文件（应该有 index.html 等）
```

如果 `dist/` 或 `public/` 是空的，说明打包失败，请截图打包窗口里的红字找开发者。

## 五、上传文件到服务器

> 🟦 **【本地电脑 → 服务器】**

### 用 FinalShell 上传

1. 在 FinalShell 已连接的窗口里，**右上角**有"文件"标签，点开
2. 左侧是你本地电脑文件，右侧是服务器文件
3. 右侧导航到 `/xmc/xihualvye/`
4. 把下面这些文件**从左侧拖到右侧**：

| 必须上传 | 本地路径 | 服务器目标路径 |
|---|---|---|
| ✅ | `server\dist\` 整个文件夹 | `/xmc/xihualvye/dist/` |
| ✅ | `server\public\` 整个文件夹 | `/xmc/xihualvye/public/` |
| ✅ | `server\package.json` | `/xmc/xihualvye/package.json` |
| ✅ | `server\package-lock.json` | `/xmc/xihualvye/package-lock.json` |
| ✅ | `server\ecosystem.config.js` | `/xmc/xihualvye/ecosystem.config.js` |

🚨 **再次提醒不能上传的文件**：
- ❌ `server\data\`（会把线上订单数据冲掉）
- ❌ `server\node_modules\`（Windows 二进制在 Linux 跑不了）
- ❌ `server\src\`（不需要）

### 上传完后，在服务器上确认

回到 FinalShell 命令行窗口（**【服务器】**）：

```bash
cd /xmc/xihualvye
ls -la
```

应该能看到：
```
dist/
public/
ecosystem.config.js
package.json
package-lock.json
logs/        ← 之前 mkdir 出来的
```

## 六、修改密码（重要！只改 1 个地方）

> 🟧 **【服务器】**

### Step 1：编辑配置文件

```bash
cd /xmc/xihualvye
vi ecosystem.config.js
```

进入 vi 编辑器后：
1. 按一下 **`i`** 键（进入编辑模式，左下角会出现 `-- INSERT --`）
2. 用方向键移动到这一行：
   ```
   ADMIN_PASSWORD: '123123',
   ```
3. 把 `123123` 改成你自己的强密码（建议 12 位以上，含字母数字符号），比如 `Xhly@2026StrongPwd`
4. 改完按 **`Esc`** 键退出编辑模式
5. 输入 **`:wq`** 然后按回车（保存并退出）

> 🆘 不小心改坏了？输入 `:q!` 然后回车，**不保存**退出，重新来过。

### Step 2：（可选）其它配置

如果有特殊需求，`ecosystem.config.js` 里还有这些可选项：

| 配置项 | 默认值 | 什么时候改 |
|---|---|---|
| `PORT` | 3456 | 想换其它端口时（同时记得改防火墙和安全组） |
| `ADMIN_USERNAME` | xhly | 想换登录账号名时 |
| `ADMIN_PASSWORD` | 123123 | **必改** |

## 九、日常更新发布（已部署后）

每次改完代码要发版，重复下面 4 步即可：

## 十一、数据库备份（强烈建议设置）

> 🟧 **【服务器】**

订单数据无价，**必须备份**。

### 自动每日备份（一次设置，长期生效）

```bash
crontab -e
```

进入编辑器后按 `i`，把下面这行粘贴进去：
```
0 2 * * * cp /xmc/xihualvye/data/xihualvye.db /xmc/backup/xihualvye_$(date +\%Y\%m\%d).db && find /xmc/backup -name "xihualvye_*.db" -mtime +30 -delete
```

按 `Esc` → 输入 `:wq` 回车保存。

效果：每天凌晨 2 点自动备份数据库到 `/xmc/backup/`，自动保留最近 30 天的备份。

### 手动备份（重要操作前）

```bash
cp /xmc/xihualvye/data/xihualvye.db /xmc/backup/xihualvye_$(date +%Y%m%d_%H%M%S).db
ls -la /xmc/backup/      # 确认备份成功
```

### 数据恢复

```bash
pm2 stop xihualvye
rm -f /xmc/xihualvye/data/xihualvye.db-wal /xmc/xihualvye/data/xihualvye.db-shm
cp /xmc/backup/xihualvye_20260512.db /xmc/xihualvye/data/xihualvye.db
pm2 start xihualvye
```

## 十二、常见问题排查（FAQ）

### Q1. 启动报错 `Cannot find module 'better-sqlite3'`
**原因**：依赖没装。
```bash
cd /xmc/xihualvye
npm install --production
pm2 restart xihualvye
```

### Q2. 启动报错 `invalid ELF` 或 `is not a valid Win32 application`
**原因**：把 Windows 的 `node_modules` 上传到 Linux 了。
```bash
cd /xmc/xihualvye
rm -rf node_modules
npm install --production
pm2 restart xihualvye
```

### Q3. `npm install` 卡住不动
**原因**：拉不到预编译包 + 没编译工具链。
```bash
sudo dnf install -y gcc gcc-c++ make           # CentOS / Rocky
# sudo apt-get install -y build-essential      # Ubuntu / Debian
npm config set registry https://registry.npmmirror.com
rm -rf node_modules
npm install --production
```

### Q4. 浏览器访问空白
1. F12 看 Console 报错
2. 在服务器上确认上传了 `public/`：
   ```bash
   ls /xmc/xihualvye/public/
   # 应该看到 index.html、assets/ 等
   ```
3. 强制刷新 Ctrl + F5

### Q5. 浏览器一直转圈打不开
99% 是云服务器安全组没放开 3456 端口，回看"三.Step 6"。

### Q6. 服务重启后所有用户被踢下线
正常现象。当前未配置固定的 token 密钥，重启后旧 token 会失效，**重新登录一次即可**。如果觉得每次掉线很烦，可以在 `ecosystem.config.js` 里加一行：
```js
TOKEN_SECRET: 'xxx一段64位的随机字符串',
```
（在服务器执行 `openssl rand -hex 32` 生成）

### Q7. 服务器重启后服务没自动起来
```bash
pm2 startup        # 查看提示并执行其输出的那一长串命令
pm2 save           # 保存当前服务列表
```

### Q8. 想看实时性能
```bash
pm2 monit
```

### Q9. 数据库文件锁住 `database is locked`
```bash
pm2 stop xihualvye
sleep 5
pm2 start xihualvye
```

## 十三、回滚方案（部署后发现新版有 bug）

### 出问题想退回上一版？

**前提**：升级前你已经备份了 `dist/`、`public/`、`data/`。

```bash
cd /xmc/xihualvye
pm2 stop xihualvye

# 1. 恢复数据库（如有需要）
cp /xmc/backup/xihualvye_<时间戳>.db data/xihualvye.db

# 2. 恢复代码产物
rm -rf dist public
cp -r /xmc/backup/dist_<时间戳> dist
cp -r /xmc/backup/public_<时间戳> public

# 3. 重启
pm2 start xihualvye
pm2 logs xihualvye --lines 30
```

### 怎么提前备份？

每次发版**前**，在服务器执行：
```bash
cd /xmc/xihualvye
TS=$(date +%Y%m%d_%H%M%S)
cp -r dist /xmc/backup/dist_$TS
cp -r public /xmc/backup/public_$TS
cp data/xihualvye.db /xmc/backup/xihualvye_$TS.db
echo "备份完成：$TS"
```

把这条命令记下来，发版前先跑一次。

## 十四、（可选）配置 nginx 反向代理 + HTTPS

> 这一节是 **进阶可选**。不配也能用，只是访问地址要带 `:3456`，且只能用 IP 不能用域名 + HTTPS。

### Step 1：安装 nginx

```bash
sudo dnf install -y nginx
```

### Step 2：写配置

```bash
cat > /etc/nginx/conf.d/xihualvye.conf << 'EOF'
server {
    listen 80;
    server_name your-domain.com;     # ← 改成你的域名

    location / {
        proxy_pass http://127.0.0.1:3456/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
EOF
```

### Step 3：启动 nginx

```bash
nginx -t
systemctl restart nginx
systemctl enable nginx
```

### Step 4：申请免费 HTTPS 证书

```bash
sudo dnf install -y certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

完成后浏览器访问 `https://your-domain.com` 即可。
